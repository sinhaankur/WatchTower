WatchTower App Center

Deploy Websites Across Nodes, Safely

WatchTower is a self-hosted deployment control plane for Linux/macOS/Windows environments. Use app-level deploys, SSH-based node sync, and automated security scanning in CI.

Repository Actions GHCR Package

Linux App Center Install

sudo ./install_app_center.sh
sudo systemctl status watchtower-appcenter
curl http://127.0.0.1:8000/health

Windows Install

powershell -ExecutionPolicy Bypass -File .\install_windows.ps1
powershell -ExecutionPolicy Bypass -File .\run_app_center_windows.ps1

macOS Install

./install_macos.sh
./run_app_center_macos.sh

Core API Endpoints

  • GET /health - service health
  • GET /apps - list registered apps
  • POST /apps/{app_name}/deploy - deploy an app by name
  • POST /deploy - generic deploy endpoint

Trigger app deploy from dev machine

WATCHTOWER_BASE_URL=http://server:8000 \
WATCHTOWER_TOKEN=change-me \
./deploy.sh --app website-main main

Safe-by-default Baseline

  • Protect API with strong WATCHTOWER_TRIGGER_TOKEN
  • Use restricted deploy users and narrow sudoers rules
  • Use SSH keys only; disable password auth on nodes
  • Require TLS + security headers on public web services
  • Keep rollback-capable backups before deployments

Release Channels

  • Container publish to GHCR via publish-container.yml
  • Python package publish to PyPI via publish-pypi.yml
  • Security gate via Trivy scan workflow
